# Listly Documentation

This folder contains comprehensive documentation for the Listly project. All contributors (human and AI) should familiarize themselves with these documents before making changes.

## 📚 Documentation Index

### [AI_DEVELOPMENT_GUIDE.md](./AI_DEVELOPMENT_GUIDE.md)
**For: AI assistants and new developers**

Comprehensive guide covering:
- Project overview, goals, and vision
- Monetization strategy and business model
- Technical specifications and architecture
- Domain models and data flow
- Development workflow and patterns
- Common tasks with code examples
- AI-specific guidelines and decision frameworks

**Start here** if you're an AI assistant or new to the project and need to understand the big picture.

### [IMPLEMENTATION_PLAN.md](./IMPLEMENTATION_PLAN.md)
**For: Project planning and execution**

Broad implementation roadmap covering:
- Current state assessment (what's built, what's missing)
- 4-phase development plan (32+ weeks)
- Week-by-week feature breakdown
- Resource requirements and costs
- Success metrics by phase
- Risk mitigation strategies

**Use this** for project planning, sprint planning, and tracking progress.

### [WEEK_1_TASKS.md](./WEEK_1_TASKS.md)
**For: Day-to-day development (Week 1)**

Detailed checklist for Week 1 implementation:
- Day-by-day task breakdown with subtasks
- Checkbox format for tracking progress
- Estimated time for each section
- Testing requirements
- Commit checkpoints
- Notes section for learnings

**Use this** as your working document during Week 1 development. Check off tasks as you complete them!

### [CODING_STANDARDS.md](./CODING_STANDARDS.md)
**For: All contributors**

Code conventions and architectural guidelines:
- Tooling and formatting rules

- Project structure conventions
- State management patterns with Riverpod
- Data layer guidelines
- Presentation and UI standards
- Testing expectations
- Error handling patterns

**Reference this** when writing code to ensure consistency.

## 🚀 Quick Start for Contributors

1. **Read** `AI_DEVELOPMENT_GUIDE.md` for context and architecture
2. **Review** `CODING_STANDARDS.md` for coding conventions
3. **Explore** the codebase following the patterns described
4. **Write tests** for any new functionality
5. **Run** `dart format .` and `flutter test` before committing
